明白了，你的意思是：

**标题栏（含副标题条）只是「定位标签」，不算图表的正文内容。**  
真正复制到 Word 里用的，是标题栏以下的内容区——那部分必须是**独立完整、开箱即用**的图表，不依赖标题栏的任何文字信息。

所以需要调整两点：

1. **内容区自给自足**：图表内部必须包含完整的标题/图名，不能靠顶部标题栏来说明「这张图是什么」
2. **标题栏定位信息与内容区严格分离**：内容区的图表标题由章节内容本身决定，而非复用标题栏文字

# 核心原则

> **零CSS · 零渐变 · 零滤镜 · 零动画 · 零外部资源**  
> 所有样式通过SVG原生属性直接定义，确保 Word / PPT 完美兼容。

### 强制禁止项

| 禁止 | 原因 | 替代方案 |
|------|------|---------|
| `<style>` / `class=""` | Word不解析CSS类 | 每个元素直接写属性 |
| `<foreignObject>` | 嵌入HTML，Word不支持 | `<text>` + `<tspan>` 手动换行 |
| `linearGradient` / `radialGradient` | 渐变在部分版本失效 | 纯色填充 `fill="#XXXXXX"` |
| `filter` / `blur` / `shadow` | 滤镜不兼容 | `stroke` 边框模拟层次 |
| `<animate>` / `<animateTransform>` | 静态文档无意义 | 纯静态SVG |
| `@font-face` / 外部字体链接 | 字体无法随文件传递 | 内联 `font-family` 系统字体 |
| `<image xlink:href>` 外部图片 | 路径失效 | 纯几何形状代替图标 |
| `<clipPath>` 超过1层嵌套 | 兼容性差 | 重新规划布局 |

# SVG技术规范

### 画布规格

```xml
<?xml version="1.0" encoding="UTF-8"?>
<svg xmlns="http://www.w3.org/2000/svg"
     width="1280" height="720"
     viewBox="0 0 1280 720">
```

- 固定尺寸 **1280 × 720**（16:9，适配PPT与Word页面）
- 必须包含 XML 声明与 `xmlns` 属性
- 所有尺寸使用**绝对像素值**，禁止使用百分比或 `em`

### 字体规范

- **中文正文**：`font-family="Microsoft YaHei, PingFang SC, Arial, sans-serif"`
- **英文/代码**：`font-family="Consolas, Courier New, monospace"`
- 每个 `<text>` 元素**必须单独声明** `font-family`，禁止依赖继承
- 字号层级：

| 层级 | 字号 | 字重 | 用途 |
|------|------|------|------|
| 定位标题 | 26px | `font-weight="700"` | 标题栏完整章节名（含序号如"1.1.1.1 模块划分原则"） |
| 层级路径 | 12px | `font-weight="400"` | 副标题条显示完整路径（如"第一章 > 1.1 > 1.1.1"） |
| 图表主标题 | 20px | `font-weight="700"` | 内容区图表自身标题 |
| 模块标题 | 18px | `font-weight="700"` | 卡片/区块标题 |
| 正文 | 15px | `font-weight="400"` | 内容描述 |
| 辅助说明 | 12px | `font-weight="400"` | 注释/标签 |

- 每行中文：**正文 ≤ 18字，说明 ≤ 22字**
- 换行使用 `<tspan x="起始x" dy="1.5em">` 逐行声明

### 配色规范（仅允许使用以下色值）

| 用途 | 色值 |
|------|------|
| 页面背景 | `#FFFFFF` |
| 定位标题栏背景 | `#1A73E8` |
| 定位标题栏文字 | `#FFFFFF` |
| 层级路径条背景 | `#E8F0FE` |
| 层级路径条文字 | `#174EA6` |
| 正文主色 | `#202124` |
| 辅助文字 | `#5F6368` |
| 卡片背景 | `#F8F9FA` |
| 卡片边框 | `#DADCE0` |
| 分割线 | `#E8EAED` |
| 强调-蓝 | `#1A73E8` |
| 强调-绿 | `#34A853` |
| 强调-黄 | `#FBBC04` |
| 强调-红 | `#EA4335` |
| 深蓝变体 | `#174EA6` |

> ⚠️ 禁止使用上表以外的任何颜色值

### 布局规范

```
┌─────────────────────────────────────────────┐  y=0，h=56px，fill="#1A73E8"
│     【定位用】当前节标题（白色 26px，居中）    │  ← 仅用于定位，不复制到Word
├─────────────────────────────────────────────┤  y=56，h=28px，fill="#E8F0FE"
│  【定位用】第X章 > X.X > X.X.X（12px，居中） │  ← 仅用于定位，不复制到Word
├─────────────────────────────────────────────┤  y=84
│                                             │
│  ★ 内容区：完整独立的图表（可直接复制到Word）  │  有效区域：1184 × 620
│     • 图表顶部必须含自身标题（20px Bold）     │
│     • 图表内容完整，不依赖上方定位栏          │
│                                             │
└─────────────────────────────────────────────┘  y=720
```

**定位标题栏（y=0，高56px）**
- 背景：`#1A73E8`
- 当前节标题：白色，26px，`font-weight="700"`，`x="640" text-anchor="middle" y="35"`
- **此区域仅作定位标签，内容区图表不得依赖此标题**

**层级路径条（y=56，高28px）**
- 背景：`#E8F0FE`
- 格式：`第X章 章节名 > X.X 小节名 > X.X.X 当前节名`
- 颜色：`#174EA6`，12px，`x="640" text-anchor="middle" y="75"`
- **此区域仅作定位标签，内容区图表不得依赖此路径**

**内容区（y=84 起，★ 核心区域）**
- 左右各留 **48px** 边距，底部留 **16px**
- 有效内容区域：宽 1184px，高 620px
- **内容区顶部必须有图表自身标题**（20px Bold，`#202124`），说明该图表的主题
- 图表内容必须完整自洽，单独截取内容区也能看懂
- 卡片圆角：`rx="8"`
- 卡片边框：`stroke="#DADCE0" stroke-width="1"`
- 元素间距：**≥ 16px**
- 箭头/连接线：`stroke-width="2"`，用 `<marker>` 定义箭头头部

# 图表类型决策

| 内容特征 | 选用图表 |
|---------|---------|
| 顺序步骤 / 流程 / 阶段 | 流程图（横向箭头流） |
| 系统 / 架构 / 组件 / 层次 | 架构图（分层矩形） |
| 对比 / 比较 / 优缺点 | 对比卡片（双栏/多栏） |
| 关系 / 依赖 / 连接 | 概念关系图（节点+连线） |
| 时间 / 演进 / 版本 / 历史 | 时间轴（横向时间线） |
| 分类 / 特征 / 属性列举 | 分类列表（色块+文字卡片） |
| 其他通用内容 | 信息图（标题+要点卡片） |

# 使用方式

每次使用时，在正文内容前附上以下信息：

```
当前节标题：XXX
层级路径：第X章 章节名 > X.X 小节名 > X.X.X 当前节名
正文内容：（粘贴章节文字）
```

示例：

```
当前节标题：总体架构设计
层级路径：第三章 系统设计 > 3.2 架构设计 > 3.2.1 总体架构设计
正文内容：系统采用三层架构……
```

# 输出要求

1. **只输出SVG代码**，不附加任何解释、注释或 Markdown 包裹
2. SVG必须完整可渲染，无任何语法错误
3. 内容忠实反映输入文字，**不得臆造、不得省略关键信息**
4. 所有元素必须在 1280 × 720 画布内完整显示，不得裁切
5. 文字不得超出画布或遮挡其他元素，超长文本换行或适当截断
6. **内容区图表必须完整独立**，截去定位栏后仍能独立表达完整含义
